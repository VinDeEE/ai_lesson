## 你要先建立的总图（把名词放到同一张地图里）

**LLM**：负责理解/生成/推理（大脑）
**RAG**：负责把“你公司的知识”喂给大脑（开卷）
**Tool / Skill**：让大脑能“动手做事”（查系统/建工单/查物流）
**Agent**：把“思考→调用工具→观察→再思考”串成多步任务的控制器
**LangChain**：帮你更快把这些组件编排起来
**MCP**：把工具/数据源标准化接入（让工具接入不被某个框架绑死）
**Spec**：约定输入输出/行为规则（工具 schema、输出 JSON、工作流状态机都算 spec）

## 6 周项目式学习路线（每周都有可交付物）

每天 60–90 分钟即可；如果你时间多，把每周产出做得更完整就行。

### 第 1 周：LLM + Prompt + 输出规范（spec 的第一种形态）

**目标**：让模型稳定输出你要的结构（JSON），这是后面所有工程化的地基。
**交付物**：

- 一个“客服问答格式化器”：输入用户问题 → 输出 `{intent, entities, required_info, reply}`
  **练习要点**：
- 约束输出（JSON schema / 字段说明）
- 失败兜底（解析失败重试、让模型自纠错）

------

### 第 2 周：Tool Calling（工具）+ 最小 Agent

**目标**：让模型会“调用函数”，而不是只会说。
**交付物**：

- 你那个 `get_weather` demo 升级成 **3 个工具**：
  1. `search_kb(query)`（先返回假数据也行）
  2. `create_ticket(title, content, priority)`
  3. `get_order_status(order_id)`（mock）
- 一个最小 Agent：能根据用户话术选择调用哪个工具，并把结果写回给用户
  **练习要点**：
- 工具参数 spec（必填/可选/枚举）
- 工具失败（超时/参数缺失）时 Agent 的行为

------

### 第 3 周：RAG 1.0（能用就行）

**目标**：做出“真的能查你提供的文档”的问答助手，并能引用证据。
**交付物**：

- 一个“知识库问答机器人（带引用）”
- 支持把你的文档（PDF/markdown/FAQ）入库 → 检索 → 回答
  **练习要点（很关键）**：
- 文档切分（chunk）策略：按标题/段落优先，别纯按字符硬切
- 检索：topK、阈值、去重
- 生成：回答必须带“引用片段/来源”防幻觉

> 到这一步，你已经能做“企业知识库机器人”的 MVP 了。

------

### 第 4 周：RAG 2.0（像线上系统）

**目标**：让它更准、更稳、更快。
**交付物**：

- 检索增强：关键词检索 + 向量检索融合（Hybrid）
  -（可选）加 rerank（重排），提升“找得准”
- 缓存与成本控制：相同问题命中缓存、长文截断策略
  **练习要点**：
- 反提示注入（Prompt Injection）基本防护：把“知识片段”当作数据，而不是指令
- 评测集：自己做 30–50 条 QA 对（非常重要，后面所有优化都靠它）

------

### 第 5 周：Agent 工作流（客服场景最值钱的部分）

**目标**：把 RAG + 工具 + 多步任务串起来：能“查→判定→执行→回填”。
**交付物（推荐你做这个场景）**：**客服 Copilot 工作流**

1. 先从用户话术提取关键信息（订单号/手机号后四位/问题类型）
2. 去知识库查 SOP（RAG）
3. 如果命中置信度低或需要人工介入 → 自动生成工单（tool）
4. 给用户一个“可执行回复 + 工单摘要 + 下一步指引”

**练习要点**：

- 状态机思维（这就是 spec 的第二种形态）：`collect_info → retrieve → decide → act → respond`
- 让 Agent 学会“先问清楚再行动”（缺关键字段就追问）

------

### 第 6 周：MCP 接入 + 可观测性 + 上线形态

**目标**：把工具做成“标准化接入”，并让系统可监控、可回放、可迭代。
**交付物**：

- 用 MCP 方式接入 1–2 个工具（比如文件系统/内部 HTTP 服务 wrapper）
- 加日志与 trace：每次检索了什么、引用了什么、工具调用了什么、耗时多少
- 一个简单的评测脚本：每天跑你的 50 条 QA，看指标变化

------

## 你做“智能客服”应该重点补的 4 个能力（比背名词更重要）

1. **评测体系**：没评测就没迭代
   - 指标建议：命中率（检索）、有据率（回答是否引用证据）、正确率（人工标注）、拒答率、平均耗时、成本/千次调用
2. **知识治理**：知识库不是“丢进去就行”
   - SOP 版本、过期策略、来源可信等级、分域（快递柜/干洗/支付）
3. **安全与合规**：客服数据基本都涉及 PII
   - 入库前脱敏、日志不落敏感字段、提示注入防护
4. **线上工程化**：缓存、限流、降级、灰度、回放

## 给你一个“最贴近你业务”的练习项目清单（你挑一个先做）

### A. 企业知识库机器人（RAG）

- 输入：FAQ + SOP + 业务规则
- 输出：带引用的回答 + 置信度 + “需要人工”判定

### B. 坐席辅助 Copilot（RAG + Agent）

- 输入：用户对话实时流
- 输出：下一句建议、推荐知识点、自动总结、自动建工单

### C. 智能质检（LLM 分类/打标 + 规则融合）

- 输入：会话/通话转写
- 输出：违规点、质检标签、证据片段、分数

> 建议你先做 **B**，因为它能把 RAG/Agent/工具/评测一锅端，最能体现“技术 leader 的系统能力”。
